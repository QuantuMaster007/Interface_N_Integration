# Failure Triage Workflow

1) **Containment first** (safety, prevent damage)
2) Classify severity (Sev-1/2/3)
3) Capture evidence:
   - logs, IO state, screenshots, wiring photos (if allowed)
   - versions from release lock
4) Assign owner + ETA
5) Decide path:
   - rollback / hotfix / accept risk / re-test
6) Update RTM and/or ICD if interface changes are required
